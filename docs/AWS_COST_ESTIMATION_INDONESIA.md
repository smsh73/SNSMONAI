## 3. DR (Disaster Recovery) 구성

### 3.1 DR 필요성에 대한 의견

#### SNS 데이터 보존 필요성
```
┌─────────────────────────────────────────────────────────────────┐
│ 질문: SNS 피드를 저장해야 하는가? SLM을 만들어야 하는가?        │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│ [권장] SNS 원본 데이터 저장이 필요한 이유:                      │
│                                                                 │
│ 1. 법적/규제 요구사항                                           │
│    - 인도네시아 정부 기관의 경우 데이터 보존 의무 가능          │
│    - 감사(Audit) 목적의 증거 자료                               │
│                                                                 │
│ 2. AI/ML 모델 개선                                              │
│    - 인도네시아 특화 감성 분석 모델 Fine-tuning                 │
│    - 위기 탐지 정확도 향상을 위한 학습 데이터                   │
│    - 트렌드 패턴 분석 및 예측 모델 개발                         │
│                                                                 │
│ 3. 히스토리컬 분석                                              │
│    - 과거 위기 상황 분석 및 리포트                              │
│    - 시계열 트렌드 분석                                         │
│    - 벤치마크 데이터                                            │
│                                                                 │
│ [참고] SLM (Small Language Model) 구축 시:                      │
│    - 인도네시아어 특화 감성 분석 모델                           │
│    - 최소 100GB ~ 1TB 학습 데이터 필요                          │
│    - Fine-tuning 비용: 별도 산정 필요                           │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

#### DR 구성 권장 여부

| 시나리오 | DR 필요성 | 권장 구성 |
|----------|----------|----------|
| 실시간 모니터링만 필요 | 낮음 | S3 백업만으로 충분 |
| 정부 기관 서비스 | 높음 | Multi-AZ + Cross-Region 백업 |
| AI 모델 학습 계획 있음 | 중간 | S3 + Glacier 아카이빙 |

### 3.2 DR 최소 구성 비용

#### Option A: 백업 기반 DR (권장 - 최소 비용)

```
┌─────────────────────────────────────────────────────────────────┐
│                    Primary Region (Jakarta)                      │
│  ┌─────────┐  ┌─────────┐  ┌─────────┐                         │
│  │   EC2   │  │   RDS   │  │  Redis  │                         │
│  │ Cluster │  │ Primary │  │ Cluster │                         │
│  └────┬────┘  └────┬────┘  └────┬────┘                         │
│       │            │            │                               │
│       └────────────┼────────────┘                               │
│                    │                                            │
│              ┌─────┴─────┐                                      │
│              │  S3 Bucket │ ← 자동 백업 (Daily)                 │
│              │  (Primary) │                                      │
│              └─────┬─────┘                                      │
│                    │                                            │
└────────────────────┼────────────────────────────────────────────┘
                     │ Cross-Region Replication
                     ▼
┌─────────────────────────────────────────────────────────────────┐
│                  DR Region (Singapore)                          │
│              ┌───────────┐                                      │
│              │ S3 Bucket │ ← 복제본 저장                        │
│              │  (Backup) │                                      │
│              └───────────┘                                      │
│                                                                 │
│  * 평시: 인프라 미가동 (비용 없음)                              │
│  * 재해 시: CloudFormation으로 인프라 자동 생성                 │
│  * RTO: 4-8시간 / RPO: 24시간                                   │
│                                                                 │

```
┌─────────────────────────────────────────────────────────────────┐
